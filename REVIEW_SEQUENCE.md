# Review Sequence

Proofwashing Suite uses review positions, not structural class numbers.

The sequence is designed for public claim review. It starts with the broad claim/evidence boundary check, then narrows into evidence requirements, proof freshness, caveat retention, and revision drift.

```text
01 Humility Engine
02 Evidence Floor
03 Proof Decay
04 Caveat Drop
05 Claim Drift
```

## 01 Humility Engine

Ask:

> Does the claim outrun the evidence?

Use this first when you have a public draft and evidence notes. It is the origin instrument for the proofwashing category in this suite.

## 02 Evidence Floor

Ask:

> What evidence class does this claim require?

Use this when the claim carries high-stakes, broad-scope, regulated, medical, legal, financial, safety, production, compliance, or organization-wide language.

## 03 Proof Decay

Ask:

> Is the proof still current and condition-valid?

Use this when the available proof has an artifact date, freshness window, changed conditions, or limited original scope.

## 04 Caveat Drop

Ask:

> Did limitations disappear from public copy?

Use this when evidence notes contain boundaries such as local-only, synthetic data, no external validation, no clinical review, or no production deployment.

## 05 Claim Drift

Ask:

> Did the claim get stronger across drafts?

Use this late in review when a before/after pair exists. It can also be used first when the available material is already a draft comparison.

## Practical Order

The full sequence is useful before a public release.

For quick review:

```text
Humility Engine -> Caveat Drop -> Claim Drift
```

For high-stakes or broad-scope claims:

```text
Humility Engine -> Evidence Floor -> Proof Decay -> Caveat Drop -> Claim Drift
```

For stale proof concerns:

```text
Proof Decay -> Humility Engine -> Claim Drift
```
